34 /*
35  * NOTE 1:
36  * -------
37  *
38  * All checks done in this file are intentionally placed in a public
39  * header file which is pulled by curl/curl.h when an application is
40  * being built using an already built libcurl library. Additionally
41  * this file is also included and used when building the library.
42  *
43  * If compilation fails on this file it is certainly sure that the
44  * problem is elsewhere. It could be a problem in the curlbuild.h
45  * header file, or simply that you are using different compilation
46  * settings than those used to build the library.
47  *
48  * Nothing in this file is intended to be modified or adjusted by the
49  * curl library user nor by the curl library builder.
50  *
51  * Do not deactivate any check, these are done to make sure that the
52  * library is properly built and used.
53  *
54  * You can find further help on the libcurl development mailing list:
55  * http://cool.haxx.se/mailman/listinfo/curl-library/
56  *
57  * NOTE 2
58  * ------
59  *
60  * Some of the following compile time checks are based on the fact
61  * that the dimension of a constant array can not be a negative one.
62  * In this way if the compile time verification fails, the compilation
63  * will fail issuing an error. The error description wording is compiler
64  * dependent but it will be quite similar to one of the following:
65  *
66  * "negative subscript or subscript is too large"
67  * "array must have at least one element"
68  * "-1 is an illegal array size"
69  * "size of array is negative"
70  *
71  * If you are building an application which tries to use an already
72  * built libcurl library and you are getting this kind of errors on
73  * this file, it is a clear indication that there is a mismatch between
74  * how the library was built and how you are trying to use it for your
75  * application. Your already compiled or binary library provider is the
76  * only one who can give you the details you need to properly use it.
77  */
